# Component Patterns

## Prefer
- small reusable components
- clear prop interfaces
- server/client separation
- state close to where it is used
- accessible form controls
- composable dashboard sections

## Avoid
- giant page files with everything in one place
- client components by default
- weak validation
- duplicated fetch logic everywhere
- styling inconsistency

## UI defaults
- shadcn/ui primitives when they fit
- Lucide icons for clarity
- Recharts when a chart adds real value
- skeleton/loading states for async UI
