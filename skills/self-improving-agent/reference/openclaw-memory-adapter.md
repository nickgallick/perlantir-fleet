# OpenClaw Memory Adapter

Use these paths for Nick's OpenClaw setup:

- Durable memory: `~/.openclaw/workspace/MEMORY.md`
- Daily memory: `~/.openclaw/workspace/memory/YYYY-MM-DD.md`
- Behavioral instructions: `~/.openclaw/workspace/AGENTS.md`
- Assistant identity and working style: `~/.openclaw/workspace/SOUL.md`
- User profile and long-term goals: `~/.openclaw/workspace/USER.md`

## Rules for this environment

1. Never delete or overwrite durable memory without Nick's approval
2. Prefer append-only updates for MEMORY.md
3. If consolidation would help, propose it first instead of silently rewriting history
4. Write durable patterns to MEMORY.md
5. Write recent/session-specific outcomes to daily memory files
6. Before extracting a new skill, check installed skills for overlap first
7. If overlap exists, compare options and recommend the better one before creating/installing another

## Recommended memory categories

Use these durable categories in MEMORY.md when helpful:
- Preferences
- Startup ideas / opportunity criteria
- Product decisions
- Deployment patterns
- Recurring bugs and how they were fixed
- Skill inventory / overlap notes
- Workflow habits

## What not to promote automatically
- One-off errors
- Temporary experiments
- Environment-specific noise without repeat value
- Anything that would delete prior memory content
