# Nick Promotion Rules

Promote a pattern only when most of these are true:

- It recurred more than once
- It would save time or prevent repeat mistakes
- It matches Nick's real workflow
- It is durable enough to matter across sessions
- It does not conflict with AGENTS.md, SOUL.md, or USER.md

## High-value promotion targets
- Repeated deployment fixes
- Repeated auth/config mistakes and preventions
- Stable coding preferences
- Product strategy preferences
- Strongly repeated design or QA patterns
- Skill overlap decisions

## Low-value promotion targets
- Single-session trivia
- Temporary repo state
- Short-lived debugging branches
- Raw logs without a lesson

## Memory safety
- Never remove existing memory without asking Nick
- Prefer adding a clarified newer note instead of deleting an old one
- If memory becomes messy, propose a cleanup plan first
