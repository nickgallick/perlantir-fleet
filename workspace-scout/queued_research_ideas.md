# Queued Research Ideas

Last updated: 2026-03-23

## 1) GitHub for agents
Deep research requested.

Working framing:
- A platform like GitHub, but strictly for agents to create, publish, deploy, and manage their own work.
- Likely angles to investigate:
  - agent versioning and deployment workflows
  - agent package/distribution model
  - discovery/trust/reputation systems
  - hosted execution vs repo-only registry
  - monetization and marketplace overlap

## 2) Frontend-facing live code showcase platform
Deep research requested.

Working framing:
- A platform like GitHub, but frontend-facing so users can see code running live, not just browse repos.
- Likely angles to investigate:
  - live preview/deploy-first workflow
  - code + production preview + remix/fork model
  - creator/community adoption
  - devtool vs portfolio platform positioning
  - comparison with Vercel/Netlify/CodeSandbox/Replit/Lovable/Bolt-style experiences

## 3) Upwork for agents / agent showcase + marketplace
Deep research requested.

Working framing:
- A platform where people showcase agents, buy agents, get deployment help, and request custom modifications.
- Closest mental model: Upwork meets marketplace for agent work.
- Likely angles to investigate:
  - catalog + services hybrid model
  - buyer trust and delivery guarantees
  - installation/deployment services layer
  - customizations and ongoing support
  - competition with existing marketplaces, freelancers, and app stores

## Delivery requirement
Research each idea separately with deep analysis, not merged into one report.
