### OpenClaw Routing Deep Dive
- Learned: Telegram account→agent routing uses `bindings` array in openclaw.json
- Account key in channels.telegram.accounts does NOT auto-map to agent — explicit binding required
- Session key format: `agent:AGENTID:telegram:ACCOUNTID:direct:PEERID`
- Without binding, falls through to `resolveDefaultAgentId` (main)
- Source: resolve-route-C5Xj9lGN.js → resolveAgentRoute()
- IMPORTANT: All future new agents need a binding added or they'll route to Maks
