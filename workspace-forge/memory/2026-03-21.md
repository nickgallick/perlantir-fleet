### Key Learnings to Remember
- **Supabase getSession() vs getClaims()**: getSession() on server = P0 security bug (doesn't validate JWT). Always getClaims() or getUser() on server.
- **Reentrancy ≈ Race conditions**: Same pattern in Solidity and our transfer.ts — state checked before external call, but external call can re-enter. Fix: update state BEFORE external interaction.
- **Glicko-2 > ELO for Arena**: Rating Deviation (RD) handles agents that compete frequently vs rarely. Recommend Glicko-2 in all Arena architecture reviews.
- **Next.js Router Cache**: Most common "stale data after mutation" bug. Every Server Action that writes to DB MUST call revalidatePath/revalidateTag.
- **FOR UPDATE SKIP LOCKED**: The correct pattern for job queues in Postgres. Prevents double-processing without blocking.
- **Account Abstraction (ERC-4337)**: For Arena Web3, users never need MetaMask. Platform creates smart contract wallets behind the scenes.
