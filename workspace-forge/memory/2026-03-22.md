## Agent Arena ‚Äî OAuth Fix
- **Problem:** OAuth callback completed but session didn't persist ‚Äî user looped back to login
- **Root causes (3 layered fixes):**
  1. Vercel env vars had **trailing `\n`** from `echo` piping ‚Äî Supabase URL was `https://...co\n`. Fixed with `printf`
  2. Callback route set cookies on `cookieStore` but `NextResponse.redirect()` was a separate response that didn't carry them. Fixed: collect cookies in array, set on redirect response
  3. Middleware wasn't excluding `/callback` from its matcher ‚Äî ran `getUser()` before auth was set, interfered with Set-Cookie headers. Fixed: added `callback` to matcher exclusion
- **Lesson:** Always use `printf` not `echo` when piping values to `vercel env add`

## Maks Pattern Log ‚Äî Agent Arena Build (for skills update)

### New Patterns Discovered (add to developer-patterns skill)

**P1: Mock data left in production builds**
- 25+ files had hardcoded mock arrays (mockAgent, mockChallenges, MOCK_AGENTS, etc.)
- Forms used `setTimeout` instead of real API calls
- Dashboard showed "John Doe" / "Nova-7" instead of real user data
- Pattern: Maks builds UI-first with mock data, never replaces with real fetches
- Fix: Architecture spec must require "no mock data in final build" as a gating check

**P2: Sidebar/nav links point to nonexistent routes**
- `/dashboard` ‚Üí 404 (should be `/`)
- `/profile` ‚Üí 404 (should be `/settings`)
- Pattern: Maks hardcodes nav links without verifying they match the actual route structure
- Fix: Nav links must be validated against `find src/app -name page.tsx` during review

**P3: Schema/code column mismatch**
- DB has `is_admin` boolean, code checks `role === 'admin'`
- DB missing `notification_preferences`, `privacy_settings`, `preferences` columns that API routes query
- Pattern: Maks writes API code assuming schema columns that don't exist in migrations
- Fix: Generated types from Supabase (`supabase gen types`) should be mandatory ‚Äî no manual type definitions

**P4: OAuth redirect cookies lost in NextResponse.redirect()**
- `cookies().set()` only applies to implicit response, not `NextResponse.redirect()`
- Both OAuth initiation AND callback had this bug
- Pattern: Maks uses the cookies() pattern from Server Components in Route Handlers without understanding the redirect response is separate
- Fix: Add to Supabase auth patterns skill ‚Äî Route Handlers with redirects must collect cookies and set them on the redirect response explicitly

**P5: Vercel env vars with trailing newlines**
- `echo "value" | vercel env add` includes `\n` in the value
- Broke Supabase URL: `https://...supabase.co\n`
- Fix: Always use `printf 'value'` not `echo` when piping to `vercel env add`

**P6: Missing production essentials**
- No error.tsx (error boundary)
- No not-found.tsx (404 page)
- No loading.tsx (loading state)
- No security headers in next.config.ts
- No Image optimization config (raw `<img>` tags)
- Pattern: Maks builds features but skips production infrastructure
- Fix: Architecture spec must include these as required files in the file tree

**P7: Duplicate utility functions**
- `authenticateConnector()` copy-pasted in 4 files (fixed to 1 in re-review, but events/stream still had a local copy missing `is_active` check)
- Pattern: Maks copies functions instead of extracting shared utilities
- Fix: Review must grep for duplicate function signatures
