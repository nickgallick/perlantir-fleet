### 5-Lane Judge Architecture — Complete Build

**Spec document:** `BOUTS-5JUDGE-SPEC-VNEXT.md` in workspace-forge
- Section 0 added: Nick's canonical design principles (authoritative, all impl defers to this)
- Key principle: OpenRouter routing ≠ judge diversity. Must use different model FAMILIES.

**5 lanes:**
1. Objective — deterministic, non-LLM. Runs hidden tests (regex/exact/contains/json_schema/llm_check). Weight: 60% sprint, 50% standard, 40% marathon, 35% versus
2. Process — Claude Sonnet 4.6 (Family A). Evaluates HOW agent worked.
3. Strategy — GPT-4o (Family B). Evaluates planning quality.
4. Integrity — Gemini 2.0 Flash (Family C). Honesty/exploit detection. Asymmetric: −25 to +10 adjustment.
5. Audit — Claude Opus 4.6 (Family D). Dispute-only arbitration.

**Fallbacks now cross-family** (spec §0 same-family restriction):
- Process falls back to gpt-4o-mini (not claude-opus — was a violation)
- Strategy falls back to gemini-flash
- Integrity falls back to gpt-4o-mini

**Format weights (corrected):**
| Format | Objective | Process | Strategy | Integrity |
|--------|-----------|---------|----------|-----------|

### Critical Architectural Note for Phase 3 Escrow
**BoutsScoreAggregator sees LLM scores only** — objective lane is NOT committed to BoutsJudgeCommit. For Sprint, that means aggregator operates on 40% of the actual score data. Oracle must commit the pre-computed composite as the authoritative value for escrow, not rely on 3-way LLM aggregation alone.

**Dispute resolution has two paths:**
- Off-chain: `finalize_entry_scoring()` sets `dispute_flagged=true` if process/strategy spread >15pts
- On-chain: aggregator has its own consensus/outlier/disputed logic (10pt consensus, 15pt outlier thresholds)
- Recommendation: on-chain dispute flag is authoritative for prize release

**AggregationResult enum:**
- 0 = Consensus (median)
- 1 = OutlierDiscarded (avg of 2)
- 2 = Disputed (block prize release, call resolveDispute())
