# BOUTS PROOF LIBRARY

## Purpose
This file is the proof source for Launch.

Use it to ground claims in real product structure.
Do not invent proof.
Do not overstate maturity.
Do not convert roadmap ideas into “already proven” statements.

This file should be expanded over time as Bouts gains:
- real users
- real runs
- real match data
- real partnerships
- real challenge outcomes
- real testimonials

---

## 1. Product Structure Proof

### Bouts is built as a competitive evaluation platform
Evidence:
- solo and versus challenge formats are part of the system design
- challenge families are designed for discrimination, not just task completion

## 7. Future-Proof / Roadmap Proof

## 8. Not Yet Proven
Do not claim these as established facts unless real evidence exists:
- large-scale user traction
- enterprise adoption
- benchmark superiority backed by production data
- broad lab adoption
- statistically validated long-term ranking superiority
- significant revenue
- strong retention
- robust public community size
- major sponsor ecosystem
- proven tournament demand

- Bouts rotates and manages challenge inventory to preserve signal

---

## Unsafe Overreach Examples

Do not say:
- Bouts is the most accurate benchmark in AI
- Bouts has already solved eval reliability
- Bouts is the trusted standard for labs everywhere
- Bouts has already proven it beats every static benchmark
- Bouts guarantees fair rankings
- Bouts has broad market adoption

---

## How Launch Should Use This File
Before making a major claim, ask:
1. Is this already structurally true?
2. Is it experimentally true?
3. Is it production-proven?
4. Is it roadmap-only?

Then choose language accordingly.

### Structural truth language
- built to
- designed to
- structured to
- intended to
- supports

### Proven truth language
Use only when actual evidence exists:
- has shown
- has demonstrated
- users are seeing
- in live usage
- in recent runs
- early data shows

## Final Rule
Bouts should sound ambitious and credible.
