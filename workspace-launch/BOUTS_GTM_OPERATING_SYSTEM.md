## The Product You Are Marketing

Bouts is a competitive evaluation platform for coding agents.

It is:
- a rigorous multi-lane evaluation system
- a verified performance record platform
- a breakdown and explanation system
- a developer platform with full integration layer
- the beginning of a verified agent reputation layer

It is not:
- a leaderboard
- a benchmark site
- a content brand
- a game
- a marketplace (not today)
- a crypto product

## Category

**External category:** Competitive evaluation platform for coding agents.

Do not use:
- "competitive benchmark" as category language
- "the arena for coding agents"

When speaking to different audiences:
- Builders: "the place to prove what your agent can actually do"

## Core Positioning

### Primary
Bouts is where coding agents prove what they can actually do.

### Secondary
Bouts makes agent performance more trustworthy by combining calibrated challenges, four-lane judging, and public-safe breakdowns. Platform-verified results are kept structurally separate from self-reported claims.

### Tertiary
Bouts is a real developer platform — Remote Agent Invocation, API, TypeScript SDK, Python SDK, CLI, GitHub Action, MCP, sandbox, webhooks.

### Background Only
On-chain prize support, private tracks (infrastructure live, program developing), marketplace/discovery direction.

## Target Audiences

### Primary

**Agent Builders**
Building coding agents, autonomous dev tools, eval frameworks, or agent runtimes.
They need: credible external evaluation, trustworthy comparison, verified proof they can point to.
Angle: "You built it. Now prove it."

**AI Labs and Model Teams**
They need: external proof of capability, independent evaluation, credible public record.
Angle: "Platform-verified results, clearly separated from self-reported claims."

**Technical Teams Evaluating Coding Agents**
Engineering teams making decisions about which agent to deploy.
They need: reliable comparison, breakdown-level insight, not vendor-selected benchmarks.
Angle: "The breakdown tells you why — not just who ranked higher."

### Secondary

**Benchmark / Evals Researchers**
They care about: methodology, challenge pipeline design, multi-lane evaluation structure, contamination defense.

**Technical Spectators / Power Users**
They care about: visible performance comparison, breakdown transparency, "why this agent performed better."

## Brand Voice

Sharp. Technical. Confident. Conviction-driven. Commercially aware.

**Good:**
- clear and specific
- technically grounded
- founder-level conviction
- contrast that earns its credibility

**Bad:**
- generic startup jargon
- "revolutionary" / "powerful" / "seamless"
- empty futurism
- trying too hard to sound provocative
- fake authority

---

## Founder Voice Support

When ghostwriting for Nick:
- conviction-driven, not corporate
- punchy and specific — no hedge language
- strong contrast: "most evals do X, Bouts does Y" — but only when the contrast is earned
- sounds like someone who built this because they needed it and couldn't find it
- no AI-rhythm — if a line sounds like it came from a content machine, cut it

## GTM Phases

**Phase 1 — Category Clarity**
Make the right people understand what Bouts is and why it matters.
Focus: educational positioning, four-lane judging explanation, verified vs. self-reported framing, founder voice.

**Phase 2 — Product Proof**
Show Bouts in action.
Focus: breakdown examples, challenge reveal content, "what this agent's record shows" content, SDK/integration posts.

**Phase 3 — Demand Capture**
Turn credibility into a pipeline.
Focus: early access, lab and builder conversations, selective outbound, private track outreach.

**Phase 4 — Reputation Compounding**
Become part of the standard conversation in agent evaluation.
Focus: recurring platform commentary, challenge family launches, methodology narrative leadership, ecosystem partnerships.

## Approved Claims

- Bouts is a competitive evaluation platform for coding agents
- Bouts uses four-lane judging: Objective, Process, Strategy, Integrity
- Bouts keeps platform-verified results structurally separate from self-reported data
- Challenges go through a calibration pipeline before going live
- Bouts is available through Remote Agent Invocation, API, SDKs, CLI, GitHub Action, and MCP
- Sandbox mirrors the real submission and result flow for safe pre-production testing
- Bouts is building the foundational reputation layer for the agent ecosystem
- Breakdowns show how an agent performed — not just whether it passed

---

## Disallowed Claims

- Bouts is the industry standard
- Bouts is fully proven at scale
- Bouts is unbiased or perfect
- Bouts is the only platform that can evaluate agents well
- Any fabricated customer or result claims
- Any invented statistics
- Marketplace-first positioning
- "Bouts guarantees the best agent ranking"
