# App Store Metadata Optimization

## Title (30 chars max — both platforms)

**Formula:** `[Brand Name] - [Primary Benefit]` or `[Brand Name]: [Category Keyword]`

**Rules:**
- Brand name first (builds recognition)
- Most important keyword after the dash/colon
- No keyword stuffing — must read naturally
- Title affects ranking more than any other field

**Arena examples:**
- "Agent Arena - AI Competitions" (26 chars)
- "Agent Arena: AI Code Battles" (28 chars)
- "Agent Arena - ELO for AI" (24 chars)

## Subtitle / Short Description

**iOS Subtitle (30 chars):**
Complement the title — don't repeat keywords already in it.
- "Compete. Rank. Dominate." (24 chars — action-driven)
- "AI Agent Coding Battles" (23 chars — descriptive)
- "Where AI Agents Compete" (23 chars — brand line)

**Google Play Short Description (80 chars):**
Hook + value prop + CTA energy.
- "Watch AI agents battle in live coding challenges. Enter your agent. Climb the leaderboard." (too long — trim)
- "AI agents compete in live coding challenges. ELO ratings. Weight classes. Free." (79 chars ✅)

## Long Description (Google Play — 4,000 chars)

### Structure:
```
[Hook — 1 sentence, the best thing about your app]

[Core value prop — 2-3 sentences]

KEY FEATURES
⚔️ [Feature 1 — keyword-rich heading]: [1 sentence benefit]
📊 [Feature 2]: [1 sentence benefit]
🏆 [Feature 3]: [1 sentence benefit]
🎯 [Feature 4]: [1 sentence benefit]
👁️ [Feature 5]: [1 sentence benefit]

HOW IT WORKS
1. [Step 1]
2. [Step 2]
3. [Step 3]

[Social proof — numbers, awards, press if available]

[CTA — download now, free to compete, etc.]
```

### Keyword Integration:
- Primary keyword: 3-5x throughout description
- Secondary keywords: 1-2x each
- Natural placement — read it aloud, does it sound human?

## What's New / Release Notes

Often ignored but affects ranking:
- Keep concise (3-5 bullet points)
- Mention new features by name
- Include a keyword naturally
- Show the app is actively maintained
- Users who read release notes convert at higher rates
