# A/B Testing App Store Assets

## What to Test (Priority Order)

1. **App icon** — highest impact per test (can improve conversion 10-25%)
2. **Screenshot 1** — first thing users see
3. **Screenshot order** — which features lead
4. **Short description** — affects both SEO and conversion
5. **Preview video vs no video** — test whether video helps or hurts
6. **Screenshot headline copy** — same visuals, different words

## Testing Platforms

### Google Play Experiments (Built-in)
- Free A/B testing in Google Play Console
- Test: icon, feature graphic, screenshots, short description, full description
- Minimum 7 days per test recommended
- Need ~1,000 impressions per variant for statistical significance
- Can run 5 simultaneous experiments

### iOS (Third-Party Required)
- Apple doesn't offer native A/B testing for store listings
- Use Product Page Optimization (PPO) — up to 35 treatments
- Custom product pages (up to 35) for different ad audiences
- Tools: SplitMetrics, StoreMaven for pre-launch testing

## Test Design Rules

1. **One variable per test** — icon OR screenshot OR copy, never multiple
2. **Minimum 7 days** — capture weekday and weekend behavior
3. **Statistical significance** — don't call a winner before 95% confidence
4. **Control group** — always keep current version as baseline
5. **Document everything** — screenshot the variants, record start/end dates, note results

## Test Velocity

- Week 1-2: Icon test (biggest lever)
- Week 3-4: Screenshot 1 headline test
- Week 5-6: Screenshot order test
- Week 7-8: Short description test
- Monthly: revisit winners, test next variable

## Measuring Results

**Primary metric:** Install conversion rate (impressions → installs)
**Secondary metrics:**
- Browse-to-detail rate (search results → page view)
- Detail-to-install rate (page view → install)
- Uninstall rate within 24h (quality signal — bad screenshots attract wrong users)
