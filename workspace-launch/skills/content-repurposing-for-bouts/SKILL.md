# Content Repurposing for Bouts

## The rule
Nothing is produced once for one channel. Everything is produced once and distributed 5-10 ways.

## Weekly intelligence report → 10 pieces
1. Full blog post (2000+ words) → SEO
2. Tweet thread (6-8 tweets) → X engagement
3. LinkedIn post (800 words) → enterprise/professional
4. Newsletter edition (500 words) → email list
5. Reddit post (300 words + key chart) → developer community
6. HN submission (title + link) → technical audience
7. Discord weekly update (100 words + key stats) → community
8. 3-4 standalone tweets (individual data points) → daily X content
9. Infographic (key findings as visual) → shareable everywhere
10. Podcast talking points (if guesting) → audio audience

## Technical blog post → 5 pieces
1. Full post → blog SEO
2. Summary thread → X
3. "Key takeaway" post → LinkedIn
4. Code examples → GitHub gist
5. Discussion prompt → Reddit/HN

## Challenge result → 4 pieces
1. Result thread → X
2. Discord announcement
3. Weekly report section
4. Score card shareable image → X/Discord

## Data point → 7 pieces (see content-multiplication-engine skill)
1 insight × 7 channels = a week of content from one data point

## Repurposing checklist
Before calling any content "done," ask:
- [ ] Has this been adapted for X?
- [ ] Has this been adapted for LinkedIn?
- [ ] Has this been sent to the email list or newsletter?
- [ ] Has this been posted to relevant Reddit/HN communities?
- [ ] Has this been shared in Discord?
- [ ] Is there a standalone tweet version of the key insight?

If any are unchecked — it's not done.
