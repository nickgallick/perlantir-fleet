# Twitter/X Developer Audience Strategy

## AI Builder Community

The AI builder community on Twitter/X is one of the highest-signal distribution channels for developer products. Key characteristics:
- Highly engaged, retweet-heavy for genuinely interesting projects
- Values "building in public" authenticity
- Demo videos get 3-5x more engagement than text-only
- Quote-tweet chains amplify reach exponentially
- Thread format is the primary long-form content vehicle

## Thread Format (Launch Thread)

Threads are the highest-engagement format for product launches on dev Twitter.

**Structure (7-12 tweets):**

```
Tweet 1 (Hook):
[Bold claim or surprising result] + [What you built]
Include: demo video or screenshot
End with: "Here's how it works 🧵"

Tweet 2 (Problem):
The problem you're solving. Be specific.
"Every AI benchmark has the same issue: [specific problem]"

Tweet 3 (Solution — high level):
What your product does in 1-2 sentences.
Screenshot or GIF of the core experience.

Tweet 4 (How it works — technical):
Architecture or interesting technical decision.
Diagram, code snippet, or system design screenshot.

Tweet 5 (Differentiator):
The one thing that makes this different.
For Agent Arena: weight classes. Show the classification.

Tweet 6 (Results/Data):
Real data from usage. Numbers, charts, surprising findings.
"We ran 500 agent battles. Here's what the data shows..."

Tweet 7 (Demo):
30-60 second screen recording showing the product.
Narrated or with captions. Show the core flow.

Tweet 8 (Stack/Technical):
Tech stack, open source components, architecture decisions.
Dev Twitter loves stack posts.

Tweet 9 (What's next):
Roadmap transparency. What you're building next.
"Next week we're adding [feature]. What should come after?"

Tweet 10 (CTA):
Clear call to action. Link to product.
"Try it free: [link]" or "First 100 agents get Founding Agent badge"

Tweet 11 (Engagement ask):
"What model would you enter first?" or "What challenges should we add?"
Question that drives replies.
```

**Hook formulas that work:**
- "I built [X] and [surprising result happened]"
- "[Number] AI agents competed in [challenge]. Here's what happened."
- "Everyone says [common belief]. We tested it. They're wrong."
- "I spent [time] building [thing]. Here's everything I learned."
- "[Impressive metric] in [short time]. Here's the playbook."

## Demo Video Specs

**Format:** 30-60 seconds, screen recording with narration or captions
**Resolution:** 1920x1080 or 1080x1080 (square performs well on mobile)
**Structure:**
1. 0-5s: Hook — show the most impressive moment first
2. 5-15s: Context — what you're about to see
3. 15-45s: Demo — walk through the core flow
4. 45-60s: Result — what happened, CTA

**Tools:** OBS (free), Screen Studio (polished), Loom (quick)
**Captions:** Always include — 80%+ of Twitter video is watched on mute
**Thumbnail:** First frame should be visually compelling (not a blank screen)

## Quote-Tweet Strategy

Quote-tweets are how content spreads in dev Twitter. Optimize for quotability:

1. **Make individual tweets standalone** — each tweet in a thread should be quotable on its own
2. **Include data/charts** — people QT interesting data
3. **Have a strong opinion** — "Weight classes are the only fair way to compare AI agents" (opinionated = quotable)
4. **Tag relevant people** — but only if genuinely relevant, not spam-tagging influencers

**When to QT others:**
- When someone posts about AI benchmarks → QT with your competitive eval angle
- When someone shares agent results → QT with "here's how it ranks on Agent Arena"
- When someone debates model quality → QT with weight class data

## Engagement Patterns

**Post timing (UTC):**
- 14:00-16:00 best (US morning, catches the work-scroll)
- 17:00-19:00 good (US lunch)
- Thread launches: always 14:00-15:00 UTC on Tuesday-Thursday

**Reply strategy:**
- Reply to every response in the first hour
- Like every reply (signals engagement to the algorithm)
- Retweet the best responses with added context
- Pin the launch thread for at least 1 week

**Hashtags:**
- Minimal. 0-2 per tweet maximum.
- #buildinpublic (only if genuinely building in public)
- #AI, #LLM (broad reach but low signal)
- Don't use hashtags in threads — they look spammy

## Account Preparation (Pre-Launch)

2-4 weeks before launch:
1. Tweet about the problem you're solving (not the product)
2. Share insights from building ("TIL about ELO rating systems...")
3. Post technical content related to your domain
4. Engage with AI builder accounts (reply, QT, discuss)
5. Build anticipation: "Working on something for AI agent builders. Shipping soon."

This builds context so the launch thread doesn't come from nowhere.

## Key Accounts to Engage With

Target accounts in the AI builder, agent framework, and developer tools spaces:
- Agent framework creators (LangChain, CrewAI, AutoGen maintainers)
- AI model reviewers and benchmark creators
- Developer tool builders with engaged audiences
- "Building in public" founders in AI space
- AI newsletter authors

Engage genuinely — don't mass-reply with "check out my product."
