## Design Generation — Google Stitch (Primary)
- MCP Server: stitch (configured via mcporter — NOT openclaw.json, that crashes the gateway)
- API Key: stored in MCP config (never expose)
- Use for: Generating professional UI designs before building ANY frontend
- Output: HTML/CSS design references — rebuild properly in target framework
- Limits: 350 standard screens/month, 50 experimental screens/month
- Free tier — no usage costs

### When to use Stitch
- Before building ANY user-facing app, website, dashboard, or landing page
- When the user asks for something to "look good" or mentions design quality
- When building mobile apps (set layout to mobile in the prompt)
- When building web apps, dashboards, admin panels

### Available MCP Tools
- `list_projects` — List all Stitch design projects
- `get_project` — Get details of a specific project
- `list_screens` — List all screens in a project
- `get_screen_code` — Get the HTML/CSS code for a specific screen
- `get_screen_image` — Get a screenshot of a screen as base64
- `create_project` — Create a new Stitch project
- `generate_screen` — Generate a new UI screen from a text prompt
- `build_site` — Map screens to routes and build a complete site

### Design-First Build Workflow
1. Create a Stitch project for the app: `create_project`
2. Generate each major screen with detailed prompts: `generate_screen`
3. Pull the HTML/CSS for each screen: `get_screen_code`
4. Use the HTML/CSS as the design reference when building React/Next.js/React Native components
5. Extract color tokens, font choices, spacing, and component patterns from the Stitch output

### Prompt Tips for Stitch
- Always specify: theme (dark/light), accent color, target platform (mobile/web)
- Include component details: "card with status pill, avatar, timestamp"
- Reference real design patterns: "Bloomberg terminal style", "Apple-like minimal"
- For multi-screen apps, maintain consistency by referencing the first screen's style
