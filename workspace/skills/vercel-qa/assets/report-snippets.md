# Report Snippets

## Product gap wording
- "This appears to be a core product gap rather than a bug. The user journey implies this capability should exist, but no usable implementation is present."

## Bug wording
- "This is a true bug because the expected feature exists conceptually and in UI flow, but the implementation breaks during normal usage."

## UX issue wording
- "This works, but the interaction quality is weak enough that it may reduce trust or completion rate."

## Verdict shortcuts
- READY FOR USERS = no critical gaps, no major broken flows, no trust-damaging UX
- NEEDS WORK = core app usable but meaningful problems remain
- NOT READY = critical flow broken or product logic too incomplete for real users
