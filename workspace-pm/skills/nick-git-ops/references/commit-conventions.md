# Commit Conventions

## Use conventional commits
- feat
- fix
- refactor
- chore
- docs
- test
- perf
- ci

## Style rules
- keep subject line short and clear
- describe the change, not the ticket process
- avoid vague messages like "stuff", "updates", or "fixes"
- commit related changes together when possible

## Good examples
- `feat: add client onboarding form`
- `fix: prevent duplicate task submission`
- `refactor: move supabase helpers into lib`
