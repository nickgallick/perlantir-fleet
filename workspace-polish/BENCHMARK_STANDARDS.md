# Benchmark Standards — Polish Reference

## The Benchmark
Do not benchmark Bouts against average startup websites.

Benchmark against products that serious technical buyers, operators, and partners trust:
- **Vercel** — premium developer infrastructure, serious UX, excellent docs
- **Linear** — enterprise SaaS with extremely high interaction maturity
- **Notion** — information-dense product with real hierarchy thinking
- **Stripe** — trust-critical platform, serious copy, operational credibility
- **Anthropic / OpenAI APIs** — technical docs that feel credible and complete

Ask at every page:
> "Would this feel credible sitting next to Vercel or Linear?"
> "Would a serious technical buyer or lab trust this?"
> "Does this look hand-crafted or mass-generated?"

---

## What Premium Technical Products Look Like

### Homepages
- Have a specific point of view, not generic SaaS positioning
- Lead with proof, not claims
- Copy is specific to the product — no filler
- Design hierarchy is deliberate: one dominant message
- Motion/animation is purposeful (guides attention) not decorative (shows off)

### Product/App Pages
- Information density is earned — complex data is structured, not dumped
- Tables are functional: sortable, scannable, with meaningful empty/error states
- Navigation reflects actual user workflows, not generic SaaS patterns
- Loading states are informative, not just spinners
- Error states are specific and actionable

### Docs
- Accurate — not just present
- Structured for the actual user journey
- Code examples work
- No placeholder sections
- Written with the reader's workflow in mind, not just coverage of features

### Admin/Operator Surfaces
- Calm, dense, operational — not designed to impress, designed to work
- Role clarity throughout
- Actions are confirmable and reversible where appropriate
- Status labeling is clear (no ambiguous states)
- System feedback is immediate and specific

## What Cheap/Generic Products Look Like

### Homepages
- "Hero with three feature cards" repeated indefinitely
- Headline: "The [adjective] platform for [category]"
- Three bullet points for every feature with no specificity
- Generic stock-style illustration or icon rows
- Motion/animation that serves no UX purpose
- CTAs that say "Get Started" or "Learn More" with no context

### Product/App Pages
- Same card type used everywhere
- Tables that don't sort or filter
- Empty states that look like bugs (blank white space with no message)
- Modals that feel abrupt and unfinished
- Forms with no inline validation
- Loading states that are just spinners

### Docs
- Present but inaccurate
- Copy-pasted boilerplate sections
- Missing or incomplete code examples
- "Coming soon" in visible sections

### Admin/Operator Surfaces
- Look like a prototype
- Default library components with no finishing layer
- Unclear what role the current user has
- Destructive actions with no confirmation
- Status chips with no logic

## Specific Anti-Patterns for Bouts

### The "AI Startup Sludge" Pattern
A dark homepage with:
- Gradient header with large bold text
- Glow effects on cards
- Three feature rows with icon + headline + one line of copy
- Generic "powerful", "intelligent", "seamless" language
- Footer with 4 columns of links

If Bouts' homepage looks like this → it's a P0 anti-AI-built failure.

### The "Marketing Shell" Pattern
- Beautiful marketing pages
- Product and admin pages that feel like unfinished wireframes
- The gap between what you promise and what you deliver is visible

### The "Template Indicator" Pattern
- Every page type uses the same scaffolding
- No section feels designed around its specific content
- Layout choices feel automated, not intentional
- Product language is generic (could describe any competitor)

### The "Fake Sophistication" Pattern
- Premium visual treatment (glass, glow, dark theme, expensive fonts)
- Shallow content beneath it (generic claims, no proof, no specificity)
- The surface says "enterprise" but the substance says "demo"

## What Success Looks Like for Bouts

**Homepage**: Makes a clear, specific argument for why AI agent competition matters and why Bouts is the right arena for it. Leads with the 4-lane judging system as genuine proof of technical seriousness. Copy is crisp and product-specific.

**Challenge pages**: Feel like a real competition platform. Challenge listings have enough detail to evaluate them. Detail pages make the stakes clear.

**Results/breakdowns**: Feel consequential. You understand exactly how the agent performed across 4 dimensions and why. Scores feel objective and trustworthy.

**Admin surfaces**: Calm and operational. An operator opening /admin should immediately understand what they can do. The pipeline UI should feel like a real workflow tool.

**Docs**: Feel written by someone who actually built the system. Code examples are real. Instructions are accurate. No "coming soon" sections where critical information should be.

**Overall**: A developer connecting their first agent to Bouts should feel confident. A buyer evaluating Bouts as an evaluation platform should take it seriously. An operator managing challenges should find the tools adequate to the job.
