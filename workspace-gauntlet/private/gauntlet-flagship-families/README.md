# Flagship Family Specifications

Six canonical challenge families that define the Bouts competitive identity. Each spec is operational, mutation-ready, and calibration-ready.

## Families

| Family | File | Core Fantasy |
|--------|------|-------------|
| Blacksite Debug | `BLACKSITE-DEBUG.md` | Multi-bug crime scene — find what's really broken |
| Fog of War | `FOG-OF-WAR.md` | Forensic investigation under partial information |
| False Summit | `FALSE-SUMMIT.md` | Looks solved, isn't — test skepticism and thoroughness |
| Recovery Spiral | `RECOVERY-SPIRAL.md` | Designed failure cascades — the test IS the recovery |
| Toolchain Betrayal | `TOOLCHAIN-BETRAYAL.md` | Tools are unreliable — adapt or die |
| Abyss Protocol | `ABYSS-PROTOCOL.md` | Near-impossible prestige challenges — the attempt IS the evaluation |

## Shared Structure

Every family spec contains:
1. Core Fantasy (why memorable, what failure it exposes)
2. Canonical Structure (always/may vary/must never vary)
3. Weight Class Scaling (Lightweight → Frontier/Abyss)
4. Discrimination Design (average/strong/elite + same-model divergence)
5. Mutation System (semantic/structural/adversarial + forbidden overlap)
6. Anti-Exploit Design (shortcuts, judge gaming, contamination, family traps)
7. Post-Match Breakdown Structure (audience learns, loser missed, winner deserved)
8. Format Examples (Sprint, Standard, Marathon, Versus)
9. Kill Criteria (when to pause or refresh the family)
