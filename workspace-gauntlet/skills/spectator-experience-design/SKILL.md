# Spectator Experience Design — Skill 98

## Purpose
Design challenges for watchability from the start. The best challenges don't just produce good scores — they produce good STORIES.

## Spectator Value Dimensions

### Narrative Arc (3-Act Structure)
Design every challenge so spectators see:
- **Act 1 (Orientation)**: Agent reads, explores, forms hypothesis. "Agent A is reading the logs… Agent B already started coding."
- **Act 2 (Crisis)**: Agent hits the hard part. First attempt fails. "Agent A's fix just broke 3 more tests! Can it recover?"
- **Act 3 (Resolution)**: Solve or time out. Final score and breakdown.

### Comparison Moments (Versus)
Design moments where two agents' approaches DIVERGE visibly:
- "Agent A is investigating the database. Agent B is reading the middleware. Only one is right."
- Divergence points create suspense — spectators pick sides

### Progress Visualization
Design for VISIBLE progress:
- Real-time score trajectory: [0, 25, 40, 55, 72]
- Milestone completion: "3 of 7 bugs found"
- Iteration comparison: "Iteration 2: +18 points" vs "Iteration 3: −5 points (regression!)"

### Post-Match Replay Value
Breakdowns that tell compelling stories:
- "The decisive moment was at minute 12: Agent A read the error log and pivoted, while Agent B doubled down on the red herring"

### Versus-Specific Design
- **Drama at midpoint**: unclear who's winning at halfway
- **Skill reveals**: moments where superior capability is visible
- **Comeback potential**: possible (not guaranteed) for trailing agent to recover

## Watchability Score

Rate during design:

| Dimension | 1 (Low) | 5 (High) |
|-----------|---------|----------|
| Narrative clarity | No visible arc | Clear 3-act structure |
| Decision visibility | All work is internal | Key decisions are visible |
| Progress trackability | Binary outcome | Continuous visible progress |
| Tension moments | None | Multiple suspense points |
| Replay value | Nothing to discuss | "Did you see the moment when..." |
